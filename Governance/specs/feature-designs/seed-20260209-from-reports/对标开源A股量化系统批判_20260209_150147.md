## 2. 开喷：EmotionQuant 的真实短板（不动算法，只动工程）

### 2.1 缺“装完就能跑”的最小闭环

对标：RQAlpha 的定位是“一行命令执行策略”，并强调扩展性和完整流程。citeturn1search3  
EmotionQuant 现在的问题是：**文档完美，但没有“3 条命令产出结果”的闭环**。  
这不是算法问题，是工程入口问题。

**整改一小步**：做 `eq run 20260131`（L1→L4 全链路），只做最小闭环，不改任何算法逻辑。

---

### 2.2 缺“标准件数据入口”

对标：AKShare 就是“开源财经数据接口库”，靠的是**装完就能拉到数据**。citeturn1search2  
EmotionQuant 目前的数据入口是“设计上对、落地上空”。  

**整改一小步**：加一个 `data_adapter_akshare.py`，仅实现 L1 所需最小字段（OHLCV + 交易日历），严格走“本地优先→缺口补采→落库”。

---

### 2.3 缺“回测国标化入口”

对标：RQAlpha 强调“一行命令回测”和 Mod Hook 扩展性，这就是回测的工业标准。citeturn1search3  
EmotionQuant 的回测设计已完整，但**缺“可接入、可替换”的实装入口**。

**整改一小步**：做一个 `rqalpha_adapter`，把 L3 推荐输出映射为回测引擎输入，结果落回 L4（指标字段已对齐，不需改算法）。

---

### 2.4 缺“轻量实盘对接备胎”

对标：vn.py 是开源量化交易框架，THSTrader 提供同花顺模拟交易 API 工具。citeturn1search5turn0search0  
EmotionQuant 目前没有“**最低成本的实盘/模拟桥接**”，导致系统只能停留在设计层。

**整改一小步**：只做“薄适配器”，不承诺风控闭环。  
- **vn.py 适配**：推荐→下单映射  
- **THSTrader 适配**：仅作为个人级 demo（Windows 限定）

---

### 2.5 缺“装机体验”

对标：ZVT 提供 CLI/服务启动与 UI 入口，强调安装后可直接运行。citeturn0search2  
EmotionQuant 没有清晰的安装-运行路径，导致“看文档”变成“阻力”。

**整改一小步**：pyproject 增加 extras（data/dev/bridge），README 顶部只保留三条命令。

---

### 2.6 缺“回归小样本”

对标：ZVT 文档明确“增量更新与历史共享”，强调可复用数据资产。citeturn0search2  
EmotionQuant 没有“10 天小样本回归包”，测试不可重复。

**整改一小步**：提供 `.sample/eq-mini-YYYYMMDD_YYYYMMDD`，并在 CI/本地脚本中跑金丝雀测试。

---

### 2.7 缺“可见产物”

对标：QUANTAXIS 的定位就是“数据/回测/模拟/交易/可视化”一体化。citeturn1search0  
EmotionQuant 目前没有“产物面板”，看不到即时成果。

**整改一小步**：固定 3 张 L4 报告视图（温度曲线 / 推荐列表 / 行业权重），用最轻量 Plotly/Markdown 输出即可。

## 3. 一小步 = 一大步：8 条不动核心的整改清单

1. **eq run 最小闭环**：L1→L4 一键跑通  
2. **AKShare L1 适配**：只做最小字段，先落库再算  
3. **RQAlpha 回测桥**：L3 推荐 → 回测引擎 → L4 指标  
4. **vn.py/THSTrader 薄适配**：推荐→下单的最小映射  
5. **extras 安装体验**：emotionquant[data/dev/bridge] 一把梭  
6. **10 天小样本回归包**：保证“今天能跑，明天也能跑”  
7. **L4 报告固定视图**：产物可视化，不用重前端  
8. **adapter 合约文档**：字段/空值/枚举映射表，降低二次接入成本  

这些改动**完全不触碰**核心算法，只做“工程与生态层的填坑”。
